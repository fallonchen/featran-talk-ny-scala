Feature Engineering: How we get from raw features like Song and User to ML algorithm friendly features like songVectors, at scale

  • 30 million+ Songs x 140 million+ active Users
  • Need to transform features so that they are small enough to be processed in a timely manner and still useful in the model
  • Fortunately we're not trying to solve all of that in this talk, just explore how to simplify one part of the process

3 Basic Feature Transformations

  • Binarizer
  • Max-Min Scaler
  • One Hot Encoding

Binarizer

Given a threshold, set a value to 0 or 1 based on its threshold

						
							val threshold = 5000000L // 50M
							val isPopular = songs.
								map(s => if(s.numPlays >= threshold) 1.0 else 0.0)
							// isPopular: Seq[Double] = List(1.0, 1.0, 1.0, 1.0)

Min-Max Scaler

Given a min and a max, scale features to fit within that range

						
							val min = 1.0
							val max = 10.0
							// find min and max song plays in one pass
							val (songsMinPlays, songsMaxPlays) = songs
																.map(s => (s.numPlays.toFloat, s.numPlays.toFloat))
																.reduce((x, y) =>
																	(math.min(x._1, y._1), math.max(x._2, y._2)))

							val minMax = songs.map { x =>
									((x.numPlays - songsMinPlays) / (songsMaxPlays - songsMinPlays)) * (max - min) + min
								}
							//minMax: Seq[Double] = List(1.9835267513990402, 1.074578134343028, 9.999999463558197, 1.0)

One Hot Encoding

Given a set of labels for a feature, the feature is transformed into an sequence that has a 1 in the position matching its label and 0 otherwise

val genres = Seq("pop","rock","hip hop","metal")

Song("Bleed", "Meshuggah", "Metal", 8741978) becomes (0, 0, 0, 1)

One Hot Encoding

						
							// Find unique ordered labels
							val genres = songs.map(_.genre)
							 				  .map(Set(_))
											  .reduce(_ ++ _)
							                  .toSeq.sorted
							// genres: Seq[String] = ArrayBuffer(Hip Hop, Metal, Rock)

							// Calculate one hots
							val oneHots = songs.map(s => genres.map(g => if(s.genre == g) 1.0 else 0.0))

							//oneHots: List(ArrayBuffer(0.0, 0.0, 1.0), ArrayBuffer(0.0, 1.0, 0.0), ArrayBuffer(1.0, 0.0, 0.0), ArrayBuffer(0.0, 0.0, 1.0))

Notice a pattern?

  • Extract T => A e.g. Song => genre
  • Map A => B e.g. genre to Set
  • Reduce B => B e.g. Set union
  • Map B => C e.g. Set to Seq in One Hot
  • Map A => C e.g. Song => Seq

Transformer

  • Extract T => A e.g. Song => genre
  • Aggregator[A, B, C] // MapReduceMap
  • Map A => C e.g. Song => Seq

Featran

FeatureSpec

						
							class FeatureSpec[T] {
  								def required[A](f: T => A)(t: Transformer[A, _, _])
  								def optional[A](f: T => Option[A], default: Option[A] = None)(t: Transformer[A, _, _])
							}

FeatureSpec

						
							import com.spotify.featran._
							import com.spotify.featran.transformers._

							val featureSpec = FeatureSpec.of[Song]
  							   .required(_.numPlays)(Binarizer("isPopular", 500000.0))
  						       .required(_.numPlays)(MinMaxScaler("numPlay_normalized"))
  						       .optional(_.genre)(OneHotEncoder("genre"))

FeatureExtractor

						
							val featureExtractor = featureSpec.extract(songs)
							// featureExtractor: com.spotify.featran.FeatureExtractor[Seq,Song]

FeatureExtractor - CollectionType

FeatureSpec#extract[M[_]: CollectionType](input: M[T]): FeatureExtractor[M, T]

						
							trait CollectionType[M[_]] { self =>
  								def map[A, B](ma: M[A], f: A => B): M[B]
  								def reduce[A](ma: M[A], f: (A, A) => A): M[A]
  								def cross[A, B](ma: M[A], mb: M[B]): M[(A, B)] // map A with C
							}

Available for Scala collections and a few other collection types that extend this trait

FeatureExtractor#featureSettings

Save the results of the Aggregator step (map-reduce-map) for future use

	
		val settings = fe.extract(songs).featureSettings
// settings: Seq[String] = List([{"cls":"com.spotify.featran.transformers.Binarizer","name":"isPopular","params":{"threshold":"500000.0"},"featureNames":["isPopular"],"aggregators":""},{"cls":"com.spotify.featran.transformers.MinMaxScaler","name":"numPlay_normalized","params":{"min":"0.0","max":"1.0"},"featureNames":["numPlay_normalized"],"aggregators":"5642096.0,3.79732121E8,1.0"},{"cls":"com.spotify.featran.transformers.OneHotEncoder","name":"genre","params":{},"featureNames":["genre_Hip Hop","genre_Metal","genre_Rock"],"aggregators":"label:Hip+Hop,label:Metal,label:Rock"}])

FeatureExtractor#featureSettings

Use saved settings to skip Aggregator step

						
							val ews = featureSpec.extractWithSettings(songs, settings)
							// ews: FeatureExtractor[Seq,Song]

val fs2 = FeatureSpec.of[Song]

Why use Featran?

  • Clarity and flexibility in how you compose and build features
  • Many common transformers provided out of the box
  • Supports many input collection types and output feature formats
  • Ability to save intermediate stages using featureSettings
